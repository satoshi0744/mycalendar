# MyCalendar 開発・運用ガイド

このドキュメントは、MyCalendar アプリの開発手順や、今後AI（Antigravity）を使って修正を再開する際の手順をまとめたものです。

## 今後の修正手順（ワークフロー）

本アプリは、**GitHub Actions** を利用した自動デプロイ（公開）の仕組みが導入されています。
開発の基本サイクルは以下の通りです。

1. **ローカル環境でAIと一緒にコードを修正する**
2. **ローカル環境で動作確認する** (`npm run dev`)
3. **AIに指示して、GitHubへPushしてもらう**
   （※自動的に本番サイト (GitHub Pages) にデプロイされます）

---

### 1. Antigravity を使った開発の再開方法

次回またアプリを修正したくなった場合は、以下の手順で**いつでも開発を再開**できます。

1. **Antigravity を起動する**
2. **該当フォルダ（`d:\mycalendar`）を選択して開く**
3. **チャット欄から「〇〇の機能を追加して」「ここを直して」と質問・指示を出す**

これだけで、過去の会話履歴が消えていたとしても、Antigravityが現在の最新のコードを自動的に読み取り、続きからの開発を完璧にサポートします。

---

### 2. ローカルでの動作確認

AIがコードを修正したら、本番に上げる前に必ず手元でテストを行います。
（※Antigravity内のターミナル機能などを使います）

1. ターミナルで以下のコマンドを実行します。
   ```bash
   npm run dev
   ```
2. ブラウザで `http://localhost:5173` にアクセスし、機能が正しく動くか確認します。

> **【重要】Googleログイン設定について**
> Google Cloud Console の「承認済みの JavaScript 生成元」に登録した `http://localhost:5173` は、**そのまま残しておいてください**。これがないと、ローカルテスト時にログインして予定を読み込むことができなくなってしまいます。

---

### 3. 本番環境（GitHub）への自動反映 (Push)

ローカルでの動作確認が完了し、問題なければ本番にアップロードします。

1. Antigravityのチャット欄で、**「修正完了したので、GitHubにPushしておいて」** と指示を出します。
2. これで私（AI）が自動的にパソコン内のGitの機能を見つけ出し、コミットとPushを行います。**サトシさん自身で難しいGitの操作をする必要はありません！**
3. Push後、**約2〜3分待つ** と、GitHub Actionsの自動処理が完了し、インターネット上の本番環境（GitHub Pages）に最新の状態が反映されます。
